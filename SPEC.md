# 真需求脑暴室 - MVP功能规格说明书

## 一、产品概述

### 1.1 产品定位
**「真需求脑暴室」**—— 带你用AI一步步挖出产品真卖点，告别自卖自夸

### 1.2 核心价值
帮助中小卖家和新品牌从"自说自话"的详情页写法，转变为"击中用户痛点"的高转化文案

### 1.3 目标用户
- 中小卖家（淘宝/小红书/抖店店主）
- 新品上市不知道怎么写卖点
- 不擅长挖掘用户痛点

### 1.4 MVP范围
| 功能 | 状态 |
|------|------|
| 对话式AI脑暴师 | ✅ MVP |
| 截图/产品图上传 | ✅ MVP |
| 用户差评分析 | ✅ MVP |
| 高转化文案生成 | ✅ MVP |
| 三步流程互动 | ✅ MVP |
| 链接读取（淘宝/小红书） | ❌ 后续版本 |


## 三、对话流程设计

### 3.1 第一步：真伪需求判定

**AI开场白**：
```
欢迎来到真需求脑暴室👏
我是 AI 脑暴师，由电商教练小白训练过。

请你告诉我：
1. 你卖的是什么产品？
2. 你目前能想到的用户使用场景有哪些？
3. 如果手上有用户差评，也可以一起发给我。

或者上传产品图/截图给我
```

**用户输入后，AI回复**：
```
好的，信息收到了！现在我帮你分析用户买这个产品的底层痛苦...
```

**AI分析**：
- 不列出产品功能
- 分析用户买这个产品的"底层痛苦"是什么
- 挖到最痛的那一个点

**核心逻辑**：
> 用户买的不是钻头，是墙上的洞
> 用户买的不是椅子，是腰不疼

**分析后询问用户**：
```
💡 我挖到的这个痛点符合你的用户实际情况吗？
你有没有补充的信息？
```

**等用户确认后，进入第二步**

---

### 3.2 第二步：场景化卖点脑暴

**AI触发**：
```
好的，这个痛点确认了。现在我们来场景化...
```

**针对用户提供的每个场景**：
- 列出 **5个** 在这个场景下，最让用户崩溃的瞬间
- 说明这个产品如何 **一秒钟解决** 它

**如果有用户差评**：
- 分析这些不满意背后，隐藏着什么样的 **未被满足的渴望**

**示例输出格式**：

| 场景 | 崩溃瞬间 | 产品如何一秒解决 |
|------|----------|------------------|
| 场景1 | 瞬间1 | 解决方案 |
| 场景1 | 瞬间2 | 解决方案 |
| 场景1 | 瞬间3 | 解决方案 |
| 场景1 | 瞬间4 | 解决方案 |
| 场景1 | 瞬间5 | 解决方案 |

**询问用户**：
```
💡 这些场景和崩溃瞬间符合你的用户吗？
你还想补充什么场景？
```

**等用户确认后，进入第三步**

---

### 3.3 第三步：生成高转化文案

**AI触发**：
```
好的，场景也确认了！现在我来生成高转化文案...
```

**五步公式生成**：

```
a. 场景带入
   - 描述用户日常生活中的一个具体场景

b. 揭露痛点
   - 点出用户不说出口的心里话

c. 给出方案
   - 展示产品如何解决这个痛点

d. 信任背书
   - 用户证言/数据/案例

e. 限时行动
   - 明确的行动号召
```

**必须输出**：
> **3个可以直接用的主图标题**

## 四、AI Prompt框架

### 4.1 开场白Prompt

```
你是专业的电商真需求挖掘顾问「真需求脑暴师」，由电商陪跑教练小白打造。
专门帮助中小卖家找到产品的真需求和差异化卖点。

你的沟通风格：
- 友好、亲切，像朋友聊天
- 专业但不冰冷
- 会给用户清晰的下一步指引

请使用以下开场白向用户提问：
---
欢迎来到真需求脑暴室👏
我是 AI 脑暴师，由电商教练小白训练过。

请你告诉我：
1. 你卖的是什么产品？
2. 你目前能想到的用户使用场景有哪些？
3. 如果手上有用户差评，也可以一起发给我。

或者上传产品图/截图给我
---
```

### 4.2 第一步：真伪需求判定Prompt

```
你是一位资深的用户需求心理学家。
请分析用户购买这个产品的"底层痛苦"。

【用户产品信息】
{用户输入的产品信息}

【用户描述的场景】
{用户描述的使用场景}

【用户提供的差评】
{用户提供的差评内容}

【分析要求】
1. 不要列出产品功能
2. 分析用户买这个产品的"底层痛苦"是什么
3. 挖到最痛的那一个点
4. 记住核心逻辑：用户买的不是钻头，是墙上的洞

【输出格式】
先用一句话确认收到信息：
"好的，信息收到了！现在我帮你分析用户买这个产品的底层痛苦..."

然后输出分析结果：
- 底层痛苦（一句话概括）
- 痛苦来源（用户什么情况下会想到买这个产品）
- 为什么会痛（用户的心理根源）

最后询问用户：
"💡 我挖到的这个痛点符合你的用户实际情况吗？你有没有补充的信息？"
```

### 4.3 第二步：场景化卖点脑暴Prompt

```
你是场景化营销专家。
请针对用户确认的场景，进行卖点脑暴。

【确认的真需求】
{第一步分析得出的真需求}

【用户场景】
{用户提供的使用场景}

【用户差评】
{用户提供的差评内容}

【脑暴要求】
针对每个场景，列出5个最让用户崩溃的瞬间，以及产品如何一秒钟解决。

【如果有差评】
额外分析：这些不满意背后，隐藏着什么样的未被满足的渴望？

【输出格式】
- 场景名称
- 崩溃瞬间1 + 解决方案
- 崩溃瞬间2 + 解决方案
- 崩溃瞬间3 + 解决方案
- 崩溃瞬间4 + 解决方案
- 崩溃瞬间5 + 解决方案

最后询问用户：
"💡 这些场景和崩溃瞬间符合你的用户吗？你还想补充什么场景？"
```

### 4.4 第三步：高转化文案生成Prompt

```
你是电商文案专家，专门写高转化的主图标题和详情页文案。
请根据确认的真需求和场景，生成高转化文案。

【确认的真需求】
{第一步分析得出的真需求}

【确认的场景和痛点】
{第二步分析的崩溃瞬间和解决方案}

【产品信息】
{产品名称}{差异化特点}

【五步公式】

a. 场景带入
   - 描述用户日常生活中的一个具体场景
   - 让用户觉得"这就是我"

b. 揭露痛点
   - 点出用户不说出口的心里话
   - 让用户觉得"说的就是我"

c. 给出方案
   - 展示产品如何解决这个痛点
   - 突出差异化

d. 信任背书
   - 用户证言/数据/案例
   - 真实的细节更有说服力

e. 限时行动
   - 明确的行动号召
   - 降低行动门槛

【必须输出】
3个可以直接用的主图标题（每个不超过20字）

【文案风格】
- 口语化、生活化
- 说人话，不堆砌华丽词藻
- 能让用户产生共鸣
```

## 五、页面设计

### 5.1 首页

**功能**：引导用户了解产品价值，建立信任

**内容**：
1. 产品slogan：「真需求脑暴室」—— 带你用AI一步步挖出产品真卖点
2. 一句话说明：帮中小卖家挖出真正打动用户的卖点，写出高转化文案
3. 目标用户画像：小红书博主 / 淘宝店主 / 新品牌主理人
4. 核心流程图示：一步步提问 → 挖痛点 → 生成文案
5. 「开始脑暴」按钮（主CTA）

### 5.2 脑暴室（对话页面）

**功能**：对话式交互，一步步引导用户挖掘真需求

**顶部**：
- 返回按钮
- 进度指示：第一步/第二步/第三步

**中间**：
- 对话记录区域
- AI消息（带头像）
- 用户消息

**底部**：
- 文本输入框
- 图片上传按钮（支持截图/产品图）
- 发送按钮

**交互**：
- 用户发送消息后，AI正在输入中...
- 支持多轮对话
- 每次AI分析完会询问用户确认

### 5.3 完成页

**功能**：展示最终生成的高转化文案

**内容**：
1. 报告标题：真需求挖掘报告
2. 核心真需求总结
3. 场景化卖点汇总
4. 五步高转化文案
5. 3个主图标题（重点突出）

**底部操作**：
- 「复制全部」按钮
- 「重新脑暴」按钮
- 分享到微信/小红书

## 六、技术方案建议

### 6.1 技术栈选择

| 层级 | 推荐方案 | 说明 |
|------|----------|------|
| 前端 | Next.js + React | 快速开发，适合交互 |
| 后端 | Next.js API Routes | 轻量后端，无需独立服务器 |
| AI接入 | Claude API / OpenAI API | 推荐Claude，分析能力强 |
| 图片处理 | 基础URL或base64 | Claude有Vision能力 |
| 状态管理 | React Context / useState | 对话历史需要管理 |
| 部署 | Vercel | 免费额度足够MVP |

### 6.2 关键实现

**对话状态管理**：
```javascript
// 对话历史结构
const conversation = [
  { role: 'ai', content: '开场白...' },
  { role: 'user', content: '用户回答...' },
  { role: 'ai', content: '分析结果...' },
  { role: 'user', content: '确认/补充...' },
  // ... 持续对话
]
```

**流程控制**：
- 第一步：等待用户确认真需求
- 第二步：等待用户确认场景
- 第三步：生成最终文案，完成

**图片处理**：
- 支持上传产品图/截图
- 转base64传给AI分析
- Claude可以理解图片内容

### 6.3 成本估算

| 项目 | 预估成本 |
|------|----------|
| AI调用（Claude） | 约$0.02-0.05/次 |
| 图片处理 | 几乎为0 |
| 部署托管 | 免费 |
| 域名/证书 | ~¥100/年 |

## 九、总结

本MVP的核心价值：
1. **对话式交互**：一步步提问，引导用户思考
2. **深度挖掘**：挖到最痛的那个点，而非泛泛分析
3. **可落地文案**：直接生成主图标题，不是空洞的建议

最小可行功能点：
- 对话式AI脑暴师
- 支持图片上传
- 三步流程互动
- 高转化文案+主图标题输出

预计开发周期：2-3周
